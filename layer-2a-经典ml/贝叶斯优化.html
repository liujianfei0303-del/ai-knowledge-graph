贝叶斯优化

核心要点

贝叶斯优化（Bayesian Optimization）是一种面向黑箱函数的高效全局优化方法，适用于目标函数评估成本高、无梯度信息、噪声较大的场景（如超参数调优、神经网络架构搜索、材料设计和 A/B 实验调参）。核心框架包含两个组件：概率代理模型（通常是高斯过程，提供预测均值与不确定性）和采集函数（在探索与利用间权衡）。常用采集函数包括 EI（Expected Improvement，期望提升）、PI（Probability of Improvement）和 UCB（Upper Confidence Bound）。每次迭代中，采集函数在当前代理模型下找到最有价值的候选点，评估后更新代理模型。相比网格搜索和随机搜索，贝叶斯优化在低维（d < 20）且评估成本高的场景下能达到更优解且迭代次数少得多。局限是高斯过程在高维和大规模数据中计算可扩展性差。

主要内容

贝叶斯优化的核心数学框架围绕概率代理模型构建最优值的后验分布。设 f: X → R 为待优化的黑箱函数，高斯过程 GP(μ(x), k(x, x’)) 提供了 f 的函数空间先验。给定观测后，任意点 x 的预测分布为高斯分布：后验均值 μₜ(x) = kₜ(x)⊤(Kₜ + σ²I)^{-1} yₜ，后验方差 σ²ₜ(x) = k(x,x) - kₜ(x)⊤(Kₜ + σ²I)^{-1} kₜ(x)。期望提升采集函数 EI(x) = E[max(f(x) - f*, 0)] 在 GP 假设下具有闭式解：EI(x) = (μₜ(x) - f*)·Φ(Z) + σₜ(x)·φ(Z)，Z = (μₜ(x) - f*) / σₜ(x)，φ 和 Φ 分别为标准正态的 PDF 和 CDF。EI 自然地平衡了探索（σₜ(x) 大的区域）与利用（μₜ(x) 高的区域）。UCB 采集函数 α_UCB(x) = μₜ(x) + κ·σₜ(x) 通过 κ 参数显式控制探索-利用权衡。

在实践中，贝叶斯优化的效率高度依赖变体选择。热启动（Warm Starting）利用先验知识初始化 GP 超参数可加速收敛；各向异性核函数（如 ARD RBF）通过自动相关检测学习各维度的特征重要性，在高维搜索空间中自动降低无关维度的影响。针对高维和多目标场景的扩展包括：随机嵌入贝叶斯优化（REMBO）将搜索空间投影到低维子空间；多任务贝叶斯优化（Multi-Task BO）复用相关任务的信息；熵搜索类方法（PES, MES）从信息论角度选择降低 f* 不确定性的候选点，在高维问题中展现更优性能。

贝叶斯优化是 AutoML 体系的核心引擎：超参数优化（Hyperopt、Optuna、GPyOpt 等框架的核心算法）、神经网络架构搜索（NAS）中作为比进化算法更高效的搜索策略、材料科学中优化化学反应条件和新材料配比（每次实验成本可达数千美元使得样本效率至关重要）。在 AI 领域的深层连接中，贝叶斯优化的大语言模型应用正在兴起——通过贝叶斯优化自动设计 prompt（如 InstructZero）、探索 LLM 的高效微调参数组合（LoRA rank、learning rate 等）。主动学习（Active Learning）与贝叶斯优化共享采集函数框架，核心区别仅在于优化目标是模型性能而非函数极值。

贝叶斯优化是一种利用高斯过程代理模型和采集函数对昂贵黑箱函数进行顺序全局最优搜索的方法。
