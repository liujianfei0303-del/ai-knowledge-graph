vLLM

核心要点

vLLM 是一个高性能的开源 LLM 推理引擎，由 UC Berkeley 的 Kwon 等人于 2023 年提出，核心创新是 PagedAttention——一种受操作系统虚拟内存分页机制启发的 KV Cache 管理算法，通过将 KV Cache 分块（block/page）并按需分配解决显存碎片和冗余存储问题。PagedAttention 将 Transformer 中自注意力层的 Key-Value Cache 划分为固定大小的块（典型 16 token/block），Attention 计算在块级别而非 token 级别进行，通过块表（block table）实现逻辑页到物理页的映射——类似操作系统将虚拟地址映射到物理内存页的机制。这一设计带来三个关键优势：接近零的内部碎片（99%+ 显存利用率，传统方法仅 20%-40%）、按需分配的显存使用（仅在请求实际生成 token 时才分配对应 block，无需预分配最大序列长度的连续空间）、以及高效的内存共享（多个 beam search 候选或并行采样共享同一个前缀的 KV Cache block，通过 Copy-on-Write 机制管理）。vLLM 同时实现了连续批处理（Continuous Batching / In-flight Batching，在每次迭代时将不同请求的 token 拼接为动态 batch 送入 GPU 计算）和基于 Prefix Caching 的自动前缀共享加速。vLLM 兼容 OpenAI API 协议（支持 /v1/chat/completions 和 /v1/completions 接口），支持主流模型架构（Llama、Mistral、Qwen、DeepSeek、GPT-NeoX、Falcon 等），内置量化支持（GPTQ、AWQ、FP8、SqueezeLLM）和 speculative decoding。vLLM 自发布以来迅速成为 LLM 部署领域最广泛采用的开源推理引擎。

主要内容

PagedAttention 内存管理机制

PagedAttention 的核心数据结构是 KV Block Table：将每个序列的逻辑 KV Cache 划分为 N 个逻辑块（logical blocks），逻辑块通过 block table 映射到物理块（physical blocks）上。Attention 计算时，PagedAttention 在物理块上执行分块注意力计算，从物理块中读取 Key-Value 张量并与 query 计算注意力分数。Block Table 管理三种核心操作：Allocation（当序列生成新 token 且当前块已满时分配新物理块）、Deallocation（序列结束时释放其占用的物理块）、Copy-on-Write（多个序列通过共享 block table 指向相同的物理块，仅在写入时才复制，用于并行采样和 beam search 的共享前缀场景）。相比传统实现（为每个请求预分配最大序列长度的连续 KV Cache 显存），PagedAttention 将显存利用率从 20%-40% 提升至 95%+，直接实现了更大的 batch size 和更高的吞吐量。KV Cache block 的大小是最重要的超参之一——block 过小导致页表膨胀和 TLB 开销，block 过大则内部碎片增加，典型取值 16 或 32 token/block。

连续批处理与调度策略

vLLM 的调度机制在每次前向传播迭代时动态决定哪些序列参与计算。当一个请求到达时，调度器将其 prefill 阶段（对 prompt token 并行计算并填充 KV Cache）与现有序列的 decode 阶段（逐 token 自回归生成）合并为同一个 batch。调度策略采用先来先服务（FCFS）基础，并加入了等待时间优先权重——当系统负载高时，调度器优先处理等待时间较长的请求以保证公平性。vLLM 支持 Request-Level Priority（请求级优先级，用户可指定请求的 priority 权重）和 API Server 级别的 Rate Limiting（速率限制）。vLLM 的批处理并非简单的序列拼接——不同序列的长度可能差异很大，它通过动态 padding 对齐到当前 batch 中最大的 token 数，空余计算量相对可控。vLLM 2.x 版本引入了 Multi-step Scheduling——每次调度执行多个生成步而非单步，通过减少调度器-执行器之间的通信开销提升整体吞吐量。

量化、部署与生态集成

vLLM 内置多种模型压缩技术的运行时支持：FP8 量化（H100/H200 原生支持 FP8 Tensor Core，可无损压缩模型至 8-bit，吞吐量翻倍）、AWQ（Activation-aware Weight Quantization，4-bit 权重量化，精度损失 < 1%）、GPTQ（逐层 Hessian 感知量化，对 W4A16 配置支持良好）、SqueezeLLM（对 outlier 权重做混合精度保存）。vLLM 也支持了 Speculative Decoding（推测解码）——使用草稿模型快速生成候选 token 序列，再由目标模型一次性验证，实现 2-3x 的单 token 延迟下降。部署架构方面，vLLM 提供 Open-AI 兼容的 API Server（vllm serve），支持 Kubernetes 部署（Helm Chart）、Docker 容器化、Prometheus 监控指标导出。在生态集成上，vLLM 是 LangChain、LlamaIndex、Haystack 等 LLM 框架的默认推理后端之一，也是 HuggingFace TGI 和 Triton Inference Server 的组件级竞争者。在训练-对齐-部署的完整链路中，vLLM 兼容 HuggingFace 模型格式，微调后的 LoRA adapter 可以通过动态 adapter 路由加载而无需导出为独立模型文件。

vLLM 是一个基于 PagedAttention 算法的高性能 LLM 推理引擎，通过高效的 KV Cache 内存管理和连续批处理实现大语言模型的高吞吐量部署。
