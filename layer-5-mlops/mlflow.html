MLflow

核心要点

MLflow 是一个开源 MLOps 平台，由 Databricks 开发并捐赠给 Linux Foundation，以其轻量级和框架无关的设计成为最广泛采用的 ML 实验管理工具。核心四大组件：Tracking（实验和运行记录——自动记录超参数、指标、模型产物、源代码版本，支持本地文件/SQLite/PostgreSQL 后端存储，提供比较运行和可视化界面）；Projects（ML 代码打包与复用规范——通过 MLproject 文件定义环境依赖和入口点，支持 Docker/Conda 环境）；Models（模型打包标准格式与部署——将模型打包为 MLflow Model 格式，支持 MLflow Serving REST API、AWS SageMaker、Azure ML、Databricks 等部署目标）；Registry（模型版本管理与生命周期——集中存储模型版本、阶段转换（Staging → Production → Archived）、审批工作流和模型谱系追踪）。MLflow 的设计哲学是轻量级、框架无关（支持 scikit-learn/PyTorch/TensorFlow/Keras/XGBoost 等所有主流框架）、渐进式采用（只使用 Tracking 也可）。在学术界和工业界均是 ML 实验管理的事实标准，特别适合需要快速实验迭代的团队和已有基础设施轻度集成需求的场景。

主要内容

Tracking 系统的数据模型与架构

MLflow Tracking 的数据模型以 Run（单次实验执行）为核心实体，每个 Run 包含：params（键值对形式的超参数，如 learning_rate=0.001）、metrics（时间序列形式的评估指标，如 accuracy 在每个 epoch 的值）、tags（用于搜索和分类的元数据标签）、artifacts（产物的 URI 引用，支持本地文件、S3、GCS、HDFS 等存储后端）。Tracking Server 支持三种后端存储模式：local filesystem（原型开发，SQLite + file store）、SQL database（MySQL/PostgreSQL 存储 runs/metadata/params/metrics，支持并发查询和 distributed execution）、以及 Artifact Store 的云存储分离架构。MLflow 的 Metric 记录采用 append-only 时间序列模型，每次 log_metric 调用在指定 step 追加一个值，支持在同一 step 记录多个 metric 值（如 epoch 级别的训练/验证 loss）。MLflow 通过 Autologging 插件机制自动捕获主流框架（scikit-learn、TensorFlow、PyTorch、XGBoost、LightGBM）的训练参数和指标，无需手动插桩代码。

MLflow Model 格式与部署管道

MLflow Model 格式定义了一个标准化的模型打包规范：每个模型被打包为一个包含 MLmodel 元数据文件、conda.yaml/requirements.txt 环境依赖、以及序列化模型产物的目录。MLmodel 文件声明模型 flavor（框架类型，如 python_function、sklearn、pytorch、tensorflow）和 signature（输入输出的 JSON Schema 约束）。python_function (pyfunc) flavor 是所有模型的统一推理接口，通过 mlflow.pyfunc.load_model() 加载模型后调用 predict()，实现了框架无关的模型服务。MLflow 内置的 Serving 部署目标包括：本地 REST API Server（基于 Flask + Gunicorn）、AWS SageMaker（自动构建 Docker 镜像并部署到 SageMaker Endpoint）、Azure ML 和 Databricks。在模型部署流水线中，MLflow 与 CI/CD 工具（Jenkins、GitLab CI）集成，通过 Model Registry 的阶段转换（Staging → Production）触发自动部署，并通过模型谱系（Lineage）追踪从训练数据到部署模型的完整链路。

与 MLOps 生态的集成模式

MLflow 的开放性体现在与 MLOps 生态系统的深度集成：与 Optuna 联合实现实验跟踪 + 超参优化闭环（Optuna 的每个 trial 自动记录为 MLflow Run）、与 Kubeflow Pipelines 集成实现容器化 ML 工作流（MLflow 记录 KFP 中每个 step 的模型和指标）、与 Airflow 配合实现定时重训练调度。MLflow 的 Model Registry 提供了模型版本控制的协作基础——支持模型版本注释、审批流转（通过 Webhook 与 Slack/Jira 集成）、以及基于 stage 的访问控制。在 LLM 场景中，MLflow 2.x 新增了 LLM 追踪能力（prompt/response 的自动记录、token 消耗统计、LLM 评估指标如 toxicity/harmfulness），以及对 LangChain、OpenAI、HuggingFace 的 Autologging 支持，使其从传统 ML 扩展至 GenAI 工作流的管理平台。

MLflow 是一个开源的机器学习生命周期管理平台，提供实验追踪、模型打包、模型注册和部署等核心功能，支持与主流 ML 框架无缝集成。
