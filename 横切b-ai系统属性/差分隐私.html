差分隐私

核心要点

差分隐私（Differential Privacy, DP）是隐私保护计算的黄金标准形式化框架，由 Dwork、McSherry、Nissim 和 Smith 于 2006 年提出，核心思想是在对数据集的分析结果中添加适当量的噪声，使得无法通过分析结果推断任何单个个体的信息是否存在或取值。形式化定义：随机机制 M 满足 ε-差分隐私，当且仅当对任意仅相差一条记录的相邻数据集 D 和 D’，以及任意输出子集 S ⊆ Range(M)，有 Pr[M(D) ∈ S] ≤ e^ε · Pr[M(D’) ∈ S]。参数 ε（隐私预算）控制隐私保护强度——ε 越小保护越强（添加噪声越多，数据效用降低），ε 越大精度越高但隐私保护弱。典型取值 ε=1 提供强保护，ε=10 提供弱保护（实际部署中在 0.1-10 之间，取决于数据敏感度和法规要求）。实现机制的基石是 Laplace 机制和 Gaussian 机制——添加与查询敏感度（Sensitivity，单条记录变化对查询结果的最大影响）成比例的噪声。差分隐私具有优秀的组合性质：串行组合（Sequential Composition）保证 ε₁ + ε₂ + … + εₖ 的总体隐私开销，并行组合（Parallel Composition）保证 max(ε₁, ε₂, …, εₖ) 的隐私开销。差分隐私在 2016 年被 Apple 引入用户端数据采集（Emoji 使用统计、QuickType 输入预测），被 Google（RAPPOR 系统）和 US Census Bureau（2020 年人口普查）以及多家大模型训练（训练数据隐私声明）采用。

主要内容

核心机制与数学原理

Laplace 机制适用于数值查询（如平均年龄、查询计数）：M(D) = q(D) + Lap(Δq/ε)，其中 Δq 为查询 q 的敏感度（Δq = max_{D,D’} |q(D) - q(D’)|），Lap(λ) 为以 0 为中心、尺度 λ 的拉普拉斯分布（PDF: (2λ)⁻¹exp(-|x|/λ)）。Laplace 机制的核心观察是加噪量与查询敏感度成正比（多数值查询的敏感度较小）而与数据集规模仅间接相关。Gaussian 机制满足 (ε, δ)-差分隐私（允许以极小的概率 δ 违反严格 ε-DP）：M(D) = q(D) + N(0, σ²·Δq²)，其中 σ² = 2ln(1.25/δ)/ε²，δ 通常取远小于 1/N 的值（如 10⁻⁵）。Gaussian 机制在深度学习中更常用因为高斯噪声与随机梯度下降的兼容性更好。指数机制（Exponential Mechanism）适用于非数值输出（如选择最有信息量的特征）：以概率正比于 exp(ε·u(D, r)/Δu) 选择输出 r，其中 u(D, r) 为效用函数。敏感度计算是差分隐私工程设计的关键：对计数（COUNT）Δq=1，对均值（MEAN）Δq = (max-min)/n，对中位数和分位数 Δq 在病态分布下可能无界（需要引入截断或平滑版本）。

差分隐私深度学习

Abadi 等人（2016）提出的差分隐私随机梯度下降（DP-SGD）是深度学习中应用差分隐私的代表性方法。标准 SGD 在每次梯度计算时未对个体样本做保护，DP-SGD 的修改包括：(1) 梯度裁剪（Gradient Clipping）——将每个样本的梯度 g_i 按 L2 范数裁剪：g_i ← g_i / max(1, ||g_i||₂/C)，将梯度的敏感度限制在 C 内；(2) 高斯加噪——在聚合梯度上添加高斯噪声：ĝ ← 1/B (Σ g_i + N(0, σ²C²I))，σ 由隐私预算 ε、批次大小 B、数据集大小 N 和迭代轮数 T 共同决定；(3) 隐私会计——使用 Moments Accountant 方法追踪总体隐私消耗（Rényi Differential Privacy 的数值跟踪器），精确计算经过多轮迭代后累积的 ε 消耗（比朴素串行组合更紧的边界）。DP-SGD 的实践挑战：隐私预算消耗速度导致模型在中等 ε（如 ε=8）时的准确率显著低于非私有训练（尤其是大型模型上）；裁剪阈值 C 的选择影响训练稳定性（C 太大梯度加噪过少，C 太小梯度丢失方向性）；DP-SGD 在批量大小很小时效果差（高斯噪声经批次平均后不够小）。DP-SGD 是 Apple、Google 等公司部署模型训练隐私保护的基础技术。

实践场景与 AI 连接

差分隐私已在多个大型系统中有实际部署。Apple 在 macOS/iOS 中使用本地差分隐私（Local DP）采集 Emoji 使用频率、新词学习等用户统计数据——每个用户在本地加噪后再发送到服务器，服务器聚合去噪后获得群体级统计，服务器无法反推个人行为。Google 的 RAPPOR 系统在 Chrome 浏览器中收集用户设置和恶意软件信息，使用 randomized response 机制在设备端加噪。US Census Bureau 在 2020 年人口普查中使用差分隐私保护每个 respondent 的居住地、年龄、种族等信息，在公开发布统计表格之前添加精心校准的噪声。在 LLM 训练中差分隐私用于保护训练数据不被模型通过 extraction attack 泄露，OpenAI（ChatGPT 训练）、Anthropic 和 Google 均在模型训练的特定阶段使用 DP 技术（带有隐私保护的 RLHF / 隐私保护的微调），在模型能力与隐私保护间权衡。联邦学习 + 差分隐私的结合是分布式隐私保护的标准模式：联邦学习保证数据不出本地，差分隐私对模型更新加噪防止攻击者从最终模型中反推参与者的训练数据。差分隐私的落地瓶颈在于隐私预算-数据效用的根本权衡和工程实施的复杂性（噪声校准、隐私会计、审计标准），但 EU AI Act、GDPR 等法规的推进正加强差分隐私在产品中的必要性。

差分隐私是一种形式化的数据隐私保护框架，通过在数据分析结果中加入受控噪声确保无法反推个体信息，已成为 AI 隐私计算和数据发布的黄金标准。
