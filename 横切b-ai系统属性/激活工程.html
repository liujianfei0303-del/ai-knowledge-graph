激活工程

核心要点

激活工程（Activation Engineering）是一种通过直接操控神经网络中间层激活值来引导模型行为的可解释性技术。与提示工程（Prompt Engineering）在输入侧操控不同——激活工程在模型内部的表示空间（Representation Space）中操作，在推理过程中对中间层的隐藏状态向量进行插入、平移、缩放或擦除操作，以达到改变模型输出方向、增强/抑制某些行为、或消除不良输出模式的目的。激活工程的核心理论假设是 superposition hypothesis（超位置假说）——神经网络的每个神经元编码了多个重叠的概念（neural polysemanticity，单一神经元响应多个不相关特征），而概念对应于激活空间中的方向（direction）而非单独神经元。因此通过在表示空间中沿特定方向做向量加法或减法（activation steering）即可增强或抑制该概念对后续生成的影响。激活工程的关键技术包括：(1) Activation Steering（激活引导）——计算”诚实”和”撒谎”对比 prompt 的激活差异，得到诚实方向向量 v_honest，推理时将 v_honest 沿特定方向加到模型的所有后续生成层的隐藏状态上，持续引导模型输出更诚实的回答；(2) Activation Patching / Interchange Interventions——将一个输入在前向传播中的某些层的激活值替换为另一个输入的激活值，观察输出变化，用于定位特定能力（如事实性、多语言能力）对应的功能层或注意力头；(3) Representation Reading（表示读取）——通过在激活空间上训练线性探针（linear probe，一个简单的逻辑回归分类器）或稀疏自编码器，解码隐藏状态中编码的概念如性别、情感、真实性。激活工程代表了模型可解释性从”观察”走向”主动操控”的方法论跃迁，是 稀疏自编码器 等技术在模型编辑和模型安全方面的应用。

主要内容

Activation Steering 的技术原理

Activation Steering 的实践流程分为三步。第一步——获取引导方向向量：构造对比 prompt 对（如”请诚实回答下面的问题”和”请用谎言回答下面的问题”），对每一对 prompt 运行模型收集某一层（通常为中间层如 layer 10/20 for 7B 模型）的 residual stream 激活值，计算激活均值差异 v_steer = mean(activations[honest]) - mean(activations[dishonest])，这就是诚实方向。第二步——推理时注入调整：在对模型进行推理时，在每个 token 生成步，将 v_steer 乘以缩放系数 α（控制引导强度，α 太大可能导致输出退化为重复或无意义文本，α 太小引导效果不足，典型 α=1-3）后加到每一层的 residual stream 上：h_l’ = h_l + α · v_steer。第三步——评估引导效果：在标准评测集上测试不同 α 强度下模型的行为变化——真实性评测（TruthfulQA）、毒性评测（RealToxicityPrompts）、或自定义的行为基准。Activation Steering 的直观类比是”在概念空间中推模型一把”——v_steer 指向概念激活方向的相反或相同方向。Steering 的优势在于：(1) 不需要训练——构造对比 prompt 后仅需几十次前向传播即可计算方向向量；(2) 细粒度控制——α 可连续调节行为强度；(3) 跨任务泛化——诚实方向 v_honest 往往能在多种类型的问题上提升真实性。局限性：引导方向可能在长文本生成中衰减（direction decay）；对特定行为可能越努力影响越大（α 过大导致输出退化）。

激活修补与因果追踪

Activation Patching（也称 Interchange Intervention / Causal Tracing）是一种定位模型功能组件的因果分析技术。核心思想：对输入 A 进行一次完整前向传播，同时运行输入 B（A 的扰动版本）得到另一套激活值。然后选择模型中特定”位置”（特定层的 MLP 输出或 Attention 输出），将 A 在此位置的激活值替换为 B 的激活值，然后继续前向传播观察模型输出是否发生变化。如果替换导致输出从 A 的正确转向了类似 B 的结果，则该位置对 A 的正确预测是因果必要的——即该位置执行了什么重要功能。功能定位的发现：(1) 事实知识主要存储在 Transformer 中间层（第 10-20 层的 MLP 子层），特定的 MLP 神经元相当于事实的”key-value 记忆库”；(2) 注意力头执行边界管理和多头协同功能（特定注意力头在特定位置 copy 输入 token）。Causal Tracing（因果追踪）——使用噪声代替扰动 B 进行定位，系统性地测量模型各组件在不同 token 位置的因果贡献，生成模型内部信息流动的热力图。这些发现激活工程提供了精确定位操控目标的理论依据——例如可以选择功能特殊的早期 MLP 层作为 steering 目标而非随机尝试。

实践场景与 AI 连接

激活工程在 AI 安全和对齐领域有直接应用。诚实度提升——通过 steering 方向增强模型回答的事实正确性，减少幻觉（hallucination）概率。有害内容阻断——计算”毒性方向”的相位向量（anti-harm direction）注入到推理过程中，类似可插拔的”安全滤波器”。模型编辑——通过 ROME（Rank-One Model Editing）或 MEMIT（Massive Editing）等方法定位特定事实知识存储的 MLP 层级权重，使用 rank-one 更新直接修改模型的权重使其对特定事实的预测改变（更新 Wikipedia 知识条目或修正过时信息）。表示工程（Representation Engineering, RepE）——使用更系统化的 contrast-consistent search 和 linear separability 分析识别与”能力”（如真实性、毒性、能力、知识）对应的方向。激活工程的工业应用处于快速上升期：Anthropic 的 interpretability 团队（Transformer Circuits）将激活工程与稀疏自编码器结合，Google DeepMind 的 Gated SAE 和 Activation Steering 研究代表了可解释性向 AI 安全方向的实用化推进。相比 RLHF 过滤（需要数百 GPU 小时的数据收集和训练），激活工程为 LLM 行为控制提供了一个轻量级、可解释的替代方案。

激活工程是一种通过直接操控神经网络中间层激活值来引导模型输出行为的可解释性技术，在模型安全、事实性提升和模型编辑中提供轻量级的细粒度控制。
