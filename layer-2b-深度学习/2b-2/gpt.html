GPT

GPT（Generative Pre-trained Transformer）是一种基于 Transformer 解码器的自回归语言模型框架，通过大规模预训练和指令微调实现强大的文本生成与推理能力。

核心要点

GPT 采用 Transformer Decoder-only 架构，核心是 自回归生成（Autoregressive Generation）：模型逐个预测序列中的下一个 token，每个位置的输出依赖之前所有 token 的因果注意力掩码（Causal Mask），确保信息只能从左向右单向流动。训练使用标准的下一 token 预测（Next Token Prediction）损失，即交叉熵。GPT 系列的关键演进：GPT-1（1.17 亿参数，证明单向 LM 预训练的有效性）→ GPT-2（15 亿参数，展示零样本迁移能力）→ GPT-3（1750 亿参数，通过 In-Context Learning 惊艳业界）→ GPT-4（多模态 + RLHF 对齐，达到人类专家水平）。

GPT 的成功推动了两个重要认知：(1) Scaling Law——模型性能与参数量、数据量、计算量之间存在可预测的幂律关系，增大规模持续带来能力提升；(2) Chat 范式——通过指令微调（Instruction Tuning）和 RLHF 对齐，将基础语言模型转化为可对话的 AI 助手。GPT-4 引入多模态能力和长上下文窗口，并通过 System Prompt 实现可配置的行为风格。GPT 系列的主要局限包括：自回归生成导致推理延迟随序列长度线性增长、事实性幻觉、以及缺乏原生的多模态理解（GPT-4V 前）。

主要内容

自回归架构详解

GPT 的 Decoder-only 架构仅包含 Transformer 的解码器部分（无编码器交叉注意力），核心组件包括：(1) 掩码多头自注意力（Masked Multi-Head Self-Attention）——通过上三角掩码矩阵确保每个位置 i 只能关注 j≤i 的 token，实现因果（Causal）生成；(2) FFN 层——与标准 Transformer 相同的两层线性 + 激活变换；(3) LayerNorm 与残差连接——采用 Pre-LN 方案（先归一化再子层变换），训练更稳定。训练损失为标准的自回归下一个 token 预测：L=−∑t=1T​logp(xt​∣x<t​,θ)。推理时采用自回归解码：在 t 步基于已生成的前 t−1 个 token 预测第 t 个 token，将新 token 追加到输入序列后重复。

关键技术：In-Context Learning

GPT-3 展示的 In-Context Learning（ICL）能力是预训练范式的重大突破：(1) Zero-shot——直接输入自然语言指令，模型理解任务意图并执行，无需任何示例；(2) Few-shot——在提示中提供 k 个示例（输入-输出对），模型从示例中推断任务模式，无需梯度更新或架构改变。ICL 的机制解释尚未完全统一，主流观点包括：示例在模型的隐空间中激活了预训练时学习的任务相关概念和模式；或等价于隐式的梯度下降过程——注意力机制在示例上执行类似参数更新的操作。ICL 的出现使提示工程（Prompt Engineering）成为 LLM 应用的核心技能。

应用场景与 AI 连接

GPT 系列模型已发展成通用 AI 基础设施：(1) 代码生成——GitHub Copilot（基于 Codex/GPT-4）将自然语言描述转化为代码，显著提升开发者效率；(2) 对话系统——ChatGPT 的指令遵循能力使其成为智能助手的事实标准，覆盖问答、写作、翻译、摘要、角色扮演等场景；(3) 多模态理解——GPT-4V/o 扩展了视觉感知和推理能力，支持图像输入和图表理解；(4) Agent 与工具使用——GPT-4 的函数调用（Function Calling）能力使 LLM 能够调用外部 API（搜索、计算、数据库查询），构建自主决策的 AI Agent。GPT 系列的演进路径清晰展示了”参数量扩展 + 数据规模扩展 + 对齐训练”的 Scaling Law 路线。
