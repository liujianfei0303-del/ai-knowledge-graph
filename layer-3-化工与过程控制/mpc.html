MPC

模型预测控制（Model Predictive Control）是一种基于过程模型的滚动优化算法，在每个采样周期在线求解有限时域优化问题并实施反馈校正。

核心要点

MPC 的核心思想是”滚动时域优化”：在每个采样时刻，利用当前系统状态估计和过程模型，预测未来 N 个控制步长的系统输出，求解带约束的最优控制序列，但只实施第一个控制作用，下一时刻重复此过程。三个核心要素：预测模型（阶跃响应/状态空间/传递函数）、滚动优化（在线求解 QP 或 NLP）、反馈校正（用实测输出修正模型误差）。这使 MPC 天然具备处理多变量耦合、显式约束和系统滞后的能力，成为过程工业先进控制（APC）的事实标准。

MPC 与 AI/ML 有深度融合方向：(1) 经济 MPC（EMPC）——直接用经济指标替代传统设定值跟踪目标，结合 RL 处理非凸经济函数；(2) 数据驱动 MPC——用 GP、神经网络替代一阶机理模型，在模型失配严重时提供概率化预测；(3) 学习型 MPC——通过 RL/在线学习自动调整 MPC 权重和约束边界。MPC 与 RL 的对比本质上是：MPC 提供安全约束保障和可解释性，RL 在复杂动态环境中更具灵活性，两者互补而非替代。

主要内容

MPC 的数学形式化描述为：在每个采样时刻 k，给定当前状态估计 x^k​，求解有限时域优化问题 minuk​,…,uk+N−1​​∑j=0N−1​(xk+jT​Qxk+j​+uk+jT​Ruk+j​)+xk+NT​Pxk+N​，满足系统动态 xk+j+1​=Axk+j​+Buk+j​ 及输入/状态约束 umin​≤uk+j​≤umax​,xmin​≤xk+j​≤xmax​。若系统为线性且约束凸，则该问题转化为二次规划（QP），可通过内点法或主动集法高效求解。工业大规模应用中，采用 Warm-Start 策略——以上一时刻的最优解作为当前时刻迭代初始点——可大幅降低求解时间至毫秒级。稳定性保证通过终端代价矩阵 P 满足 Lyapunov 方程或终端约束集实现，确保闭环系统的渐近稳定性和递归可行性。

MPC 的主要算法分支涵盖线性 MPC、非线性 MPC（NMPC）和显式 MPC（explicit MPC）。线性 MPC 使用线性模型和二次目标函数，是工业应用最成熟的变体；NMPC 处理非线性过程模型（如聚合反应动力学模型），需在线求解非凸 NLP，计算实时性仍为挑战——实际做法采用多步打靶（multiple shooting）将动态约束离散化或 ACADO/CasADi 等代码生成工具加速求解；显式 MPC 将 QP 的解参数化为状态分区仿射函数 u∗=Fi​x+Gi​（适用于 ∀x∈CRi​），将在线优化变为查表操作，仅适用于小规模系统。此外，线性变参数 MPC（LPV-MPC）在不同操作点切换线性模型，兼顾非线性补偿和计算效率。

MPC 与 AI/ML 的融合在近年来取得突破性进展。深度 MPC 使用神经网络逼近系统动力学 xk+1​=fθ​(xk​,uk​) 作为预测模型，结合可微分优化层实现端到端的学习与控制。不确定性感知 MPC 采用高斯过程（GP）回归建模模型残差 Δ(xk​,uk​)∼GP(m(x),k(x,x′))，为预测注入概率化认知——在数据稀疏区域自动扩大置信区间，实现安全感知的保守控制。在工业应用中，主流 APC 平台（如 Honeywell Profit Controller、Aspen DMC3、Schneider DCS）的 MPC 模块已集成数据驱动模型更新能力和约束自适应调整功能，成为流程工业数字化转型的核心引擎。MPC 与 RL 的融合路径正在从学术研究向工业试点过渡，预计将在复杂批次过程和能源系统调度中产生显著价值。
